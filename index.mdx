## What is a dev container?

A **development container** lets you use a Docker container as a full-featured development environment. Instead of configuring tools on your local machine, you define everything your project needs — runtimes, dependencies, extensions, settings — in a `devcontainer.json` file that lives in your repository.

Any developer who opens the project gets the same, reproducible environment automatically.

```json .devcontainer/devcontainer.json
{
  "name": "Node.js",
  "image": "mcr.microsoft.com/devcontainers/javascript-node:22",
  "features": {
    "ghcr.io/devcontainers/features/github-cli:1": {}
  },
  "postCreateCommand": "npm install",
  "customizations": {
    "vscode": {
      "extensions": ["dbaeumer.vscode-eslint"]
    }
  }
}
```
